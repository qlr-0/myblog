2、hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

3、部署博客构建环境

  • 安装git

  • 安装 node.js

  • 部署Hexo

    Win+R-> ‘cmd’ 打开命令行

    先配置npm的安装源

    npm config set registry https://registry.npm.taobao.org

    然后再是安装hexo

4、构建个人博客

  • 在码云上创建仓库
  • 先注册一个账号，新建一个仓库

  • 在你的电脑上创建Hexo文件夹创建一个文件夹，然后到这个文件夹里右键Git Bash Here运行hexo init（这里需要等待一段时间)

  • 添加SSH公钥

    打开刚才创建的文件夹，右键Git GUI Here

    Help->Show SSH key->Generate SSH KEY 创建密钥

    复制公钥到

    设置->(左边列表)SSH公钥(粘贴公钥)

  • 本地预览博客

  • 更改博客主题

    默认使用的主题是landscape，在目录themes/landscape下，更改一个个人喜欢的主题，如hexo-theme-yilia。

    git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

    主题库：https://hexo.io/themes/

  • 在_config.yml中配置Git

    注意：冒号后面一定要有空格，否则不能正确识别。

  • 发布到Gitee

  • 配置Gitee Pages


    在/source/_posts目录下就可以写我们的博客

    每一次修改本地库都要打一遍hexo clean && hexo g && hexo d来将本地库同步到Gitee，并且在Gitee Pages更新一下。

PHP反序列化

  • PHP反序列化漏洞也叫PHP对象注入，是一个非常常见的漏洞,基本都是围绕着serialize()，unserialize()这两个函数。在运行到这两个函数时会有相应的magic方法的调用。

unserialize(): 恢复原先被序列化的变量
serialize(): 把复杂的数据类型压缩到一个字符串中 数据类型可以是数组，字符串，对象等

序列化public private protect参数

Pubic 公有

Private 私有

Protect 保护

序列化之后打印出:
O:4:"test":3:{s:4:"name";s:8:"xiaoming";s:9:"testsex";s:3:"man";s:6:"*age";s:2:"20";}

{}外

  • O代表object，还有一种情况是A，代表数组
  • 4代表对象名字占4个字符
  • 3代表对象里有三个变量

{}内

  • s的位置代表数据类型，s代表string，i代表int
  • 数据类型之后的数字代表变量的字符长度

Public属性序列化后格式: 成员名

Private属性序列化后格式: %00类名%00成员名

Protected属性序列化后的格式: %00*%00成员名

一些magic方法

  • __construct() 当一个对象创建时被调用

  • __destruct() 当一个对象销毁时被调用

  • __toString() 当一个对象被当作一个字符串使用

  • __wakeup 将在序列化之后立即被调用

  • 魔术方法：官方文档中介绍

  • 举一个别人的例子

  • 打印出:
    __construct abc __toString __sleep __wakeup __toString __destruct __destruct

  • 通过这个例子就可以清晰的看到魔法函数在符合相应的条件时便会被调用

wakeup绕过

  • 若存在__wakeup方法，调用 unserilize() 方法前则先调用__wakeup方法，但是如果序列化字符串中表示对象属性个数的值大于真实的属性个数时会跳过__wakeup的执行
  • 例如：O:4:"test":3:{s:4:"name";s:8:"xiaoming";s:9:"testsex";s:3:"man";s:6:"*age";s:2:"20";}更改{}外的3就可实现绕过

OC 绕过

正则匹配的规则是: 在不区分大小写的情况下，若字符串出现"o:数字"或者"c:数字"这样的格式就会被过滤，而serialize()的参数为object的时候，参数类型肯定为对象 O，序列化字符串的格式为参数格式:参数名长度,因此O:数字这样的字符串肯定无法通过正则匹配。这时我们可以在数字前加上一个+，即O:+数字，就可以绕过正则的过滤。

私有属性绕过

前面有说过Private属性序列化后格式，所以可以通过构造%00类名%00成员名，来提取private的内容。protected则通过构造%00*%00成员名来提取。

.htaccess文件(或者”分布式配置文件”）,全称是Hypertext Access(超文本入口)。提供了针对目录改变配置的方法， 即，在一个特定的文档目录中放置一个包含一个或多个指令的文件， 以作用于此目录及其所有子目录。作为用户，所能使用的命令受到限制。管理员可以通过Apache的AllowOverride指令来设置。

print_r()函数用于打印变量，以更容易理解的形式展示。

1.localeconv()

localeconv() 函数返回包含本地数字及货币信息格式的数组。

localeconv()函数返回以下数组元素：

  • [decimal_point] - 小数点字符
  • [thousands_sep] - 千位分隔符
  • [int_curr_symbol] - 货币符号 （例如：USD）
  • [currency_symbol] - 货币符号 （例如：$）
  • [mon_decimal_point] - 货币小数点符号
  • [mon_thousands_sep] - 货币千位分隔符
  • [positive_sign] - 正值符号
  • [negative_sign] - 负值符号
  • [int_frac_digits] - 国际小数数字
  • [frac_digits] - 本地小数数字
  • [p_cs_precedes] - if 如果货币符号在正值之前，则是 True (1)，否则是 False (0)。
  • [p_sep_by_space] - True (1) 如果货币符号与正值之间有空间，则是 True (1)，否则是 False (0)。
  • [n_cs_precedes] - True (1) if 货币符号在负值之前，则是 True (1)，否则是 False (0)。
  • [p_sep_by_space] - True (1) 如果货币符号与负值之间有空间，则是 True (1)，否则是 False (0)。
  • [p_sign_posn] - 格式化选项：
    • 0 - 在数量和货币符号周围的圆括号
    • 1 - 数量和货币符号之前的 + 号
    • 2 - 数量和货币符号之后的 + 号
    • 3 - 货币符号之前的 + 号
    • 4 - 货币符号之后的 + 号
  • [n_sign_posn] - 格式化选项：
    • 0 - 在数量和货币符号周围的圆括号
    • 1 - 数量和货币符号之前的 - 号
    • 2 - 数量和货币符号之后的 - 号
    • 3 - 货币符号之前的 - 号
    • 4 - 货币符号之后的 - 号
  • [grouping] - 显示如何分组数字的 Array（例如：3 指示 1 000 000）
  • [mon_grouping] - 显示如何分组货币数字的 Array（例如：2 指示 1 00 00 00）
